🧭 导航组件详解

导航是应用的核心功能，Flutter 提供了强大的导航系统。本教程将详细介绍 Navigator、Router 以及页面跳转的最佳实践。

💡 重要提示：Flutter 3.28+ 引入了 Navigator 2.0，提供了更强大的路由管理和状态管理能力。

🧭 Flutter 路由系统详解

路由是应用中页面跳转和导航的核心机制。Flutter 提供了两套路由系统：Navigator 1.0（命令式）和Navigator 2.0（声明式）。

📚 路由核心概念

  • Route（路由）：代表应用中的一个页面或屏幕，是屏幕的抽象
  • Navigator（导航器）：管理路由栈的组件，负责页面的入栈和出栈
  • 路由栈：由 Navigator 维护的页面堆栈，遵循后进先出（LIFO）原则
  • 页面切换：通过推送（push）和弹出（pop）操作实现页面跳转
  • 路由参数：在页面跳转时传递的数据，包括导航参数和返回参数

📊 路由栈示意图

← 返回上一页（pop）
↓
当前页面（页面 C）
上一页（页面 B）
首页（页面 A）
↓
→ 跳转到新页（push）

🔄 Navigator 1.0 vs Navigator 2.0

两套路由系统对比
特性 Navigator 1.0 Navigator 2.0
编程范式 命令式（Imperative） 声明式（Declarative）
状态管理 由 Navigator 内部管理 由开发者完全控制
适用场景 简单应用、快速开发 复杂应用、深度链接、Web
学习曲线 简单 较复杂
API Navigator.push/pop Router、RouteInformationParser
URL 同步 需要手动处理 自动同步
⚠️ 选择建议：
  • 初学者或简单应用：推荐使用 Navigator 1.0，学习曲线低，开发效率高
  • 复杂应用或需要深度链接：推荐使用 Navigator 2.0，提供更强大的控制能力
  • Web 应用：必须使用 Navigator 2.0 以支持 URL 和浏览器导航

🎯 路由类型

1. MaterialPageRoute

Material Design 风格的页面切换，包含平台特定的转场动画。

MaterialPageRoute(
  builder: (context) => const SecondPage(),
  fullscreenDialog: false,  // 是否为全屏对话框
  maintainState: true,      // 是否保持页面状态
)

2. CupertinoPageRoute

iOS 风格的页面切换，使用 iOS 原生转场动画。

CupertinoPageRoute(
  builder: (context) => const SecondPage(),
)

3. PageRouteBuilder

自定义页面转场动画的通用路由。

PageRouteBuilder(
  pageBuilder: (context, animation, secondaryAnimation) => const SecondPage(),
  transitionsBuilder: (context, animation, secondaryAnimation, child) {
    const begin = Offset(0.0, 1.0);
    const end = Offset.zero;
    const curve = Curves.ease;

    var tween = Tween(begin: begin, end: end).chain(CurveTween(curve: curve));
    var offsetAnimation = animation.drive(tween);

    return SlideTransition(
      position: offsetAnimation,
      child: child,
    );
  },
)

📤 路由参数传递

通过构造函数传递参数

// 定义带参数的页面
class DetailsPage extends StatelessWidget {
  final String title;
  final int id;

  const DetailsPage({
    super.key,
    required this.title,
    required this.id,
  });

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text(title)),
      body: Center(child: Text('ID: $id')),
    );
  }
}

// 传递参数
Navigator.push(
  context,
  MaterialPageRoute(
    builder: (context) => DetailsPage(
      title: '产品详情',
      id: 123,
    ),
  ),
);

通过 arguments 传递参数（命名路由）

// 在 MaterialApp 中定义路由
MaterialApp(
  routes: {
    '/': (context) => const HomePage(),
    '/details': (context) {
      final args = ModalRoute.of(context)!.settings.arguments as Map;
      return DetailsPage(
        title: args['title'] as String,
        id: args['id'] as int,
      );
    },
  },
);

// 传递参数
Navigator.pushNamed(
  context,
  '/details',
  arguments: {'title': '产品详情', 'id': 123},
);

页面返回数据

// 推送页面并等待返回结果
final result = await Navigator.push(
  context,
  MaterialPageRoute(builder: (context) => const SelectionPage()),
);

if (result != null) {
  print('用户选择了: $result');
}

// 在 SelectionPage 中返回数据
Navigator.pop(context, '选择的结果');

🛡️ 路由守卫和中间件

路由守卫用于在页面跳转前进行权限验证、登录检查等操作。

class AuthGuard {
  static Future canAccess(BuildContext context, String route) async {
    // 检查用户是否已登录
    final isLoggedIn = await AuthService.isLoggedIn();
    
    if (!isLoggedIn && route != '/login') {
      // 未登录，跳转到登录页
      Navigator.pushReplacementNamed(context, '/login');
      return false;
    }
    
    return true;
  }
}

// 使用路由守卫
class SafeNavigator {
  static Future pushNamed(
    BuildContext context,
    String routeName, {
    Object? arguments,
  }) async {
    final canAccess = await AuthGuard.canAccess(context, routeName);
    if (!canAccess) return null;
    
    return Navigator.pushNamed(context, routeName, arguments: arguments);
  }
}

// 使用方式
SafeNavigator.pushNamed(context, '/profile');

🔗 深度链接

深度链接允许用户通过 URL 直接打开应用中的特定页面。

MaterialApp(
  // 支持深度链接
  onGenerateRoute: (settings) {
    // 解析 URL
    final uri = Uri.parse(settings.name!);
    
    // 处理不同路径
    switch (uri.path) {
      case '/':
        return MaterialPageRoute(builder: (_) => const HomePage());
      case '/product':
        final id = int.tryParse(uri.queryParameters['id'] ?? '0');
        return MaterialPageRoute(
          builder: (_) => ProductPage(id: id ?? 0),
        );
      case '/user':
        final userId = uri.queryParameters['userId'];
        return MaterialPageRoute(
          builder: (_) => UserProfilePage(userId: userId),
        );
      default:
        return MaterialPageRoute(builder: (_) => const NotFoundPage());
    }
  },
  
  // 处理外部链接
  initialRoute: '/',
);

// 示例 URL：
// https://example.com/product?id=123
// https://example.com/user?userId=john

💡 路由最佳实践

  • 使用命名路由：集中管理路由，避免硬编码页面路径
  • 路由常量化：将路由路径定义为常量，避免字符串拼写错误
  • 参数类型安全：使用数据模型类传递复杂参数，而不是 Map
  • 错误处理：为未知路由定义 404 页面
  • 页面缓存：使用 AutomaticKeepAliveClientMixin 保持页面状态
  • 转场动画：根据平台选择合适的PageRoute类型
  • 路由测试：编写单元测试验证路由跳转逻辑
  • 性能优化：使用 const 构造函数减少重建

📍 1. Navigator 基础导航

Navigator 是 Flutter 的核心导航组件，管理页面栈。

基本导航方法

// 推送新页面（不返回数据）
Navigator.push(
  context,
  MaterialPageRoute(builder: (context) => const SecondPage()),
);

// 返回上一页
Navigator.pop(context);

// 推送新页面并等待结果
final result = await Navigator.push(
  context,
  MaterialPageRoute(builder: (context) => const SelectionPage()),
);
print('选择结果: $result');

// 返回时传递数据
Navigator.pop(context, '返回的数据');

// 替换当前页面
Navigator.pushReplacement(
  context,
  MaterialPageRoute(builder: (context) => const NewPage()),
);

// 清空栈并推送新页面
Navigator.pushAndRemoveUntil(
  context,
  MaterialPageRoute(builder: (context) => const HomePage()),
  (route) => false,
);

命名路由

// 定义路由
MaterialApp(
  routes: {
    '/': (context) => const HomePage(),
    '/details': (context) => const DetailsPage(),
    '/settings': (context) => const SettingsPage(),
  },
);

// 导航到命名路由
Navigator.pushNamed(context, '/details');

// 带参数的命名路由
Navigator.pushNamed(
  context,
  '/details',
  arguments: {'id': 123, 'title': '产品详情'},
);

// 在目标页面获取参数
final args = ModalRoute.of(context)!.settings.arguments as Map;

🔧 2. 路由构造函数

使用 onGenerateRoute 实现更灵活的路由管理。

MaterialApp(
  onGenerateRoute: (settings) {
    switch (settings.name) {
      case '/':
        return MaterialPageRoute(builder: (_) => const HomePage());
      case '/details':
        final id = settings.arguments as int;
        return MaterialPageRoute(
          builder: (_) => DetailsPage(id: id),
        );
      default:
        return MaterialPageRoute(
          builder: (_) => const NotFoundPage(),
        );
    }
  },
);

📑 3. TabBar 和 BottomNavigationBar

实现应用内的标签导航。

TabBar 示例

class TabBarExample extends StatefulWidget {
  const TabBarExample({super.key});

  @override
  State createState() => _TabBarExampleState();
}

class _TabBarExampleState extends State
    with SingleTickerProviderStateMixin {
  late TabController _tabController;

  @override
  void initState() {
    super.initState();
    _tabController = TabController(length: 3, vsync: this);
  }

  @override
  void dispose() {
    _tabController.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('TabBar 示例'),
        bottom: TabBar(
          controller: _tabController,
          tabs: const [
            Tab(icon: Icon(Icons.home), text: '首页'),
            Tab(icon: Icon(Icons.search), text: '搜索'),
            Tab(icon: Icon(Icons.person), text: '我的'),
          ],
        ),
      ),
      body: TabBarView(
        controller: _tabController,
        children: const [
          HomePage(),
          SearchPage(),
          ProfilePage(),
        ],
      ),
    );
  }
}

BottomNavigationBar 示例

class BottomNavExample extends StatefulWidget {
  const BottomNavExample({super.key});

  @override
  State createState() => _BottomNavExampleState();
}

class _BottomNavExampleState extends State {
  int _currentIndex = 0;

  final List _pages = [
    const HomePage(),
    const SearchPage(),
    const ProfilePage(),
  ];

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: _pages[_currentIndex],
      bottomNavigationBar: BottomNavigationBar(
        currentIndex: _currentIndex,
        onTap: (index) {
          setState(() {
            _currentIndex = index;
          });
        },
        items: const [
          BottomNavigationBarItem(
            icon: Icon(Icons.home),
            label: '首页',
          ),
          BottomNavigationBarItem(
            icon: Icon(Icons.search),
            label: '搜索',
          ),
          BottomNavigationBarItem(
            icon: Icon(Icons.person),
            label: '我的',
          ),
        ],
      ),
    );
  }
}

🎬 6. 页面过渡动画

自定义页面切换的过渡动画。

// 自定义页面过渡
Navigator.push(
  context,
  PageRouteBuilder(
    pageBuilder: (context, animation, secondaryAnimation) =>
        const SecondPage(),
    transitionsBuilder: (context, animation, secondaryAnimation, child) {
      const begin = Offset(1.0, 0.0);
      const end = Offset.zero;
      const curve = Curves.ease;

      var tween = Tween(begin: begin, end: end).chain(
        CurveTween(curve: curve),
      );

      return SlideTransition(
        position: animation.drive(tween),
        child: child,
      );
    },
    transitionDuration: const Duration(milliseconds: 300),
  ),
);

// 淡入淡出效果
Navigator.push(
  context,
  PageRouteBuilder(
    pageBuilder: (context, animation, secondaryAnimation) =>
        const SecondPage(),
    transitionsBuilder: (context, animation, secondaryAnimation, child) {
      return FadeTransition(
        opacity: animation,
        child: child,
      );
    },
  ),
);
